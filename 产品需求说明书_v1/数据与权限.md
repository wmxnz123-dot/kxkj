﻿# 数据与权限（口径与矩阵）V1.0

本文件用于统一数据实体口径、字段定义、脱敏策略、权限矩阵，避免原型与研发阶段理解不一致。

## 1. 核心数据实体（建议）
### 1.1 用户（User）
- userId：平台用户唯一标识
- realName：姓名
- idType：证件类型
- idNo：证件号码
- mobile：手机号
- communityId / communityName：所属社区
- realNameStatus：实名状态（未实名/已实名）

### 1.2 数据目录（Catalog）
- catalogId：目录ID
- parentId：父目录ID
- name：目录名称
- code：目录编码（全局唯一）
- sort：排序
- enabled：启用状态
- description：目录说明（用户端提示）

### 1.3 数据项（DataItem）
- dataItemId：数据项ID
- catalogId：所属目录
- title：数据项名称
- fields：字段集合（key/value）
- sourceSystem：来源系统
- sourceDepartment：来源部门（可选）
- updatedAt：更新时间
- claimStatus：认领状态（未认领/已认领/异常）
- correctStatus：纠错状态（无/进行中/已更正/已驳回）

### 1.4 认领记录（ClaimRecord）
- claimId：认领记录ID
- userId
- catalogId / dataItemId
- action：认领/取消（V1 先不做取消，可预留）
- createdAt
- operatorType：用户/系统

### 1.5 纠错工单（CorrectionTicket）
- ticketId：工单号
- userId
- dataItemId / fieldKey：纠错对象
- originValue：原值
- correctedValue：修正值
- reason：原因（枚举/文本）
- attachments：附件列表
- status：待审核/需补充/已通过/已驳回
- auditOpinion：审核意见
- createdAt / updatedAt
- auditedAt（可选）

### 1.6 自有数据（UserUpload）
- uploadId
- userId
- catalogId
- templateId
- title：标题（可由模板字段组合生成）
- fields：结构化字段
- attachments
- status：正常/已删除
- createdAt / updatedAt

### 1.7 授权模板（AuthTemplate）
- templateId
- scenarioName：场景名称
- scenarioCode：场景编码
- scope：授权范围（目录/字段集合）
- purpose：用途说明
- validityPolicy：有效期策略
- riskTips：风险提示文案
- revokeTips：撤销说明文案
- enabled

### 1.8 授权记录（AuthRecord）
- authId：授权编号
- userId
- requesterName：被授权方名称
- requesterType：类型（机构/部门/企业）
- scenarioCode
- scopeSnapshot：授权范围快照（不可变）
- purposeSnapshot：用途快照（不可变）
- validFrom / validTo
- status：有效/已撤销/已过期/已拒绝
- decidedAt：用户决策时间
- revokedAt（可选）

### 1.9 数据服务（DataService）
- serviceId
- name
- description
- entryUrl / route
- requiredScope：所需授权范围（建议关联模板）
- enabled
- sort
- recommended：是否推荐

## 2. 字段展示与脱敏策略（默认）
### 2.1 用户端（本人查看）
- 证件号/手机号：允许明文展示（本人视角）
- 对外授权：默认仅展示“范围摘要”，详细字段在确认页展开

### 2.2 管理端（默认脱敏）
- 证件号：仅显示前2后4（示例：37********1234）
- 手机号：仅显示前三后四（示例：139****2222）
- 明文查看：需要权限点“敏感信息明文查看”，并记录操作日志
- 导出：需要权限点“数据导出”，导出策略默认脱敏；明文导出需额外权限点

## 3. 权限矩阵（建议）
### 3.1 用户端权限
- 已实名：认领/纠错/上传/授权/撤销授权
- 未实名：仅浏览有限内容；敏感操作统一拦截并引导实名

### 3.2 管理端权限点（示例）
- USER_SEARCH：用户检索
- USER_PROFILE_VIEW：查看用户档案
- SENSITIVE_VIEW：敏感信息明文查看
- DATA_EXPORT：导出数据
- CATALOG_CONFIG：配置认领目录与字段映射
- UPLOAD_TEMPLATE_CONFIG：配置自有目录与上传模板
- AUTH_TEMPLATE_CONFIG：配置授权模板
- AUTH_AUDIT_VIEW：查看授权记录审计
- CORRECTION_REVIEW：纠错工单审核处理
- SERVICE_CONFIG：数据服务配置

### 3.3 角色与权限建议映射
- 超级管理员：全量
- 配置管理员：CATALOG_CONFIG、UPLOAD_TEMPLATE_CONFIG、AUTH_TEMPLATE_CONFIG、SERVICE_CONFIG、USER_SEARCH、USER_PROFILE_VIEW
- 审核员：CORRECTION_REVIEW、USER_SEARCH、USER_PROFILE_VIEW
- 审计员：AUTH_AUDIT_VIEW、USER_SEARCH（可选）、DATA_EXPORT（可选，需按合规控制）

## 4. 状态枚举（建议）
### 4.1 认领状态（claimStatus）
- UNCLAIMED：未认领
- CLAIMED：已认领
- ERROR：异常（源不可用/权限不足/数据缺失）

### 4.2 纠错工单状态（ticketStatus）
- PENDING：待审核
- NEED_MORE：需补充
- APPROVED：已通过
- REJECTED：已驳回

### 4.3 授权状态（authStatus）
- PENDING：待处理
- GRANTED：有效（已同意）
- REVOKED：已撤销
- EXPIRED：已过期
- DENIED：已拒绝
